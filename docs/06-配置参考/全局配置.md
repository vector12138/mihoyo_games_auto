# 全局配置参考

## global 配置项
| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| use_gpu | boolean | true | 是否使用GPU加速OCR识别 |
| max_retries | int | 3 | 操作失败最大重试次数 |
| retry_delay | int | 1 | 重试间隔时间（秒） |
| click_delay | float | 0.2 | 点击操作后延迟时间（秒） |
| auto_shutdown | boolean | false | 任务完成后是否自动关机 |
| wol_mode | string | "auto" | WOL唤醒检测模式：<br>- `auto`：自动检测模式，通过系统事件和唤醒源判断<br>- `force_on`：强制判定为WOL唤醒，任务结束强制自动关机<br>- `force_off`：强制判定为本地开机，永远不会自动关机 |
| debug | boolean | false | 是否开启调试模式，开启后会输出更详细的日志 |

## 步骤配置字段说明
每个任务步骤支持以下配置字段：
| 字段 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| name | string | 步骤{序号} | 步骤名称，用于日志和通知显示 |
| type | string | 必填 | 步骤类型，支持：click、screenshot、ocr_match、wait、input、key_press、start_app、close_app、run_cmd 等 |
| fail_act | string | "continue" | 步骤失败后动作：<br>- `continue`：继续执行后续步骤<br>- `stop`：终止流程，后续仅执行标记为`force: true`的步骤 |
| force | boolean | false | 是否强制执行：前面步骤失败后，只有标记为强制的步骤仍会执行 |
| timeout | int | 30 | 步骤执行超时时间（秒） |
| retries | int | 继承全局max_retries | 单步骤自定义重试次数，不设置则使用全局配置 |

## Telegram 配置
| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| enabled | boolean | false | 是否启用Telegram通知和控制 |
| api_url | string | "http://127.0.0.1:8080" | Telegram Bridge服务API地址 |
| api_key | string | "" | API访问密钥，需要和bridge服务配置一致 |
| listen_chat_ids | array | [] | 监听的聊天ID列表，为空则监听所有允许的聊天 |
| poll_interval | int | 1 | 消息轮询间隔（秒） |
| command_prefix | string | "/" | 命令前缀，识别消息是否为控制命令 |

## 游戏通用配置
| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| window_mode | string | "fullscreen" | 游戏窗口模式：fullscreen（全屏）/windowed（窗口化） |
| resolution | string | "1920x1080" | 游戏分辨率，推荐使用1920x1080以获得最佳识别效果 |
